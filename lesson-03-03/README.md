# 基本情報技術者試験に出るアルゴリズム

## 1. アルゴリズムの種類
基本情報技術者試験で出題される主なアルゴリズムは以下の3種類です。

- **整列アルゴリズム**（データを並べるアルゴリズム）
- **探索アルゴリズム**（データを見つけるアルゴリズム）
- **再帰的アルゴリズム**（処理の途中で自分自身を呼び出すアルゴリズム）

---

## 2. 整列アルゴリズム（ソートアルゴリズム）
データを特定の順番に並べるアルゴリズムです。

### 主な整列アルゴリズム
- **バブルソート**  
  - 隣り合ったデータを比較し、順序が異なっていたら交換を繰り返す。

- **選択ソート**  
  - 配列の先頭から順に最小値を選択し、先頭のデータと交換を繰り返す。

- **挿入ソート**  
  - 整列済みの部分に未整列のデータを適切な位置に挿入する。

- **クイックソート**（試験でよく出題）  
  - **基準値**（ピボット）を選び、  
    - 小さいグループ  
    - 大きいグループ  
    に分ける。  
  - この手順を繰り返し、分割できなくなると昇順に整列される。

---

## 3. 探索アルゴリズム
特定のデータを見つけ出すアルゴリズムです。

### 主な探索アルゴリズム
- **線形探索法**  
  - 先頭から順に1つずつ照合する単純な方法。

- **ハッシュ法**（試験でよく出題）  
  - ハッシュ関数を使い、データをハッシュ値に変換して検索する。  
  - **ハッシュ表**に格納し、高速な探索を可能にする。  
  - **デメリット**：「衝突」が発生する可能性がある（同じハッシュ値が発生）。

- **2分探索法**（試験でよく出題）  
  - **整列済みのデータ**を対象に、探索範囲を半分に分けながら検索する。  
  - **特徴**:
    1. データを **昇順または降順** に整列させておく必要がある。
    2. 探索範囲を **2つに分割** しながら探索する。

---

## 4. 再帰的アルゴリズム
処理の途中で **自分自身を呼び出す** アルゴリズムです。

### 再帰的アルゴリズムの特徴
- **再帰呼び出し**：関数が自分自身を呼び出す。
- **再帰関数**：再帰的な処理を行う関数。

### 主な問題（試験でよく出題）
1. **総和（シグマ Σ）**  
   - 与えられた数のすべての合計。
   
2. **剰余（mod 演算）**  
   - 割り算のあまりを求める。  
   - 例：`5 mod 2 = 1`

3. **階乗（n!）**  
   - `n! = n × (n-1) × ... × 1`  
   - 例：`3! = 3 × 2 × 1 = 6`

### アルゴリズムの表現方法
アルゴリズムは以下の4つの方法で表現される。
1. **文章**
2. **流れ図**
3. **数式**
4. **プログラム言語**

---

**基本情報技術者試験では、特にクイックソート、ハッシュ法、2分探索法、再帰的アルゴリズムがよく出題されるため、重点的に学習することが推奨される。**


# 再帰呼び出しとアルゴリズムの計算量

## 1. 再帰呼び出しとは？
**再帰呼び出し（Recursion）** とは、関数の中で **自分自身を呼び出して処理を実行すること** です。

例えば、1から `n` までの合計を求めるとき、  
「`n` に `n-1` までの合計を足せばいい」という考え方を使います。

---

## 2. 再帰的アルゴリズムとは？
**再帰的アルゴリズム（Recursive Algorithm）** とは、  
問題をより小さな部分問題に分け、同じ手順を繰り返す方法です。

特によく出題されるのは以下の3つです。
- **総和の計算**（1から `n` までの合計）
- **剰余の計算**（最大公約数の求め方など）
- **階乗の計算**（`n!` の計算）

例えば、`n!`（nの階乗）は  
`n! = n × (n-1)!` という形で表すことができます。

---

## 3. 再帰関数の練習の重要性
再帰的な計算は、一つひとつのステップを手で書き出して理解するとよいです。  
例えば `f(5)` の計算を考えると、次のような流れになります。

f(5) = 5 + f(4) = 5 + (4 + f(3)) = 5 + 4 + (3 + f(2)) = 5 + 4 + 3 + (2 + f(1)) = 5 + 4 + 3 + 2 + 1 = 15


このように、**再帰の流れを紙に書いて確認することが大切** です。

---

## 4. アルゴリズムの計算量（オーダー）
### **オーダー（計算量）とは？**
アルゴリズムを完了するために必要な **計算の回数（計算量）** を示します。  
データ数と計算量の関係を表す記法を **オーダー記法（Big-O記法）** といいます。

### **オーダー記法の例**
| 記法       | 計算量の増え方        | 例 |
|------------|-----------------|------|
| O(1)       | 一定            | 配列の先頭要素を取得 |
| O(log n)   | ゆっくり増加      | 二分探索 |
| O(n)       | データ数に比例    | ループ処理 |
| O(n log n) | 少し速く増加      | 高速ソート |
| O(n²)      | 急激に増加        | 二重ループ |

---

## 5. 2分探索法のオーダー
**2分探索（Binary Search）** とは、データを **半分に分けながら探す方法** です。  
データの数が `n` のとき、  
**探索のたびに `n` を `1/2` にしていくため、計算量は O(log n) になります。**

例えば、`n = 16` のデータがある場合：

1回目 → 8個に減る 2回目 → 4個に減る 3回目 → 2個に減る 4回目 → 1個になる（終了）

**4回（log₂ 16 = 4）で探索が完了する** ことがわかります。

---

## 6. 対数（log）の計算方法
### **log とは？**
**log（対数）** とは、指数を求めるための方法の1つです。

### **対数の基本**
\[
\log_b(a) = c
\]
（bを何回かけるとaになるか？）

例：
- **\(\log_2(8) = 3\)**  （2を3回かけると8）
- **\(\log_3(9) = 2\)**  （3を2回かけると9）

2分探索の計算量 **O(log n)** は、この「対数」の考え方を使って求められます。

---

## **まとめ**
- **再帰とは？** → 自分自身を呼び出す関数のこと
- **再帰的アルゴリズムの例** → 総和、剰余、階乗
- **オーダー記法（計算量）** → O(1), O(log n), O(n), O(n log n), O(n²)
- **2分探索の計算量** → O(log n)（データを半分に分ける）
- **log とは？** → 何回かけると特定の数になるかを求める方法
